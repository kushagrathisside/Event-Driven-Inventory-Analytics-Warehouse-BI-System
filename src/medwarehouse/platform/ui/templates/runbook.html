Recovery Priorities

1. Restore infrastructure first

If PostgreSQL or Kafka is down, recover those before starting Bronze, Silver, or warehouse jobs.

2. Check freshness before replaying work

Use stage lag and artifact timestamps to see where the pipeline actually stopped moving.

3. Clear quality issues before publishing trust

Non-zero quarantine or quality-check counts indicate correctness problems, not just execution problems.

4. Prefer bounded jobs for repair

Run `inventory_silver`, `inventory_stage`, `load_inventory_facts`, and `refresh_views` in that order when catching up.

Suggested Repair Flow

  1. Start local services if the infrastructure page reports Kafka or PostgreSQL as stopped.
  2. Confirm the warehouse becomes reachable and the DAG file is still present.
  3. Run or restart `Inventory Bronze Stream` only after Kafka is healthy.
  4. Run `Inventory Silver Refresh` to validate and rewrite Silver outputs.
  5. Run `Stage Inventory Events`, `Load Inventory Facts`, and `Refresh Semantic Views` to rebuild warehouse visibility.
  6. Finish with `Quality Checks` and verify alert volume falls back to zero or expected noise.
